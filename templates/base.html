Dynamical Systems

So much in this world unfolds over time. And whenever it does, it can be described by a dynamical system – that is one of the most fundamental insights of science. All that is needed is to describe a state of the things studied in numbers and then find rules for how the state will change in the next instant of time. For example, the spheres Galileo threw from the Leaning Tower of Pisa could be described by their height and velocity. The rule says both spheres will accelerate according to the gravitational constant which is the same for every object on Earth. That is why they both touched the ground at the same time. The beauty lies in the fact, that if one can do this for one system, the same pattern of behaviour can be found occurring again and again in nature.

Usage Guide

Choosing System

Feel free to choose any of the systems from the dropdown menu. You will see how the system evolves in time, you can read its description and ponder the parameters that define its dance.

Phase & Cartesian Views

Each system is represented in two ways. Phase view shows, as time progresses, the relative position of the first chosen function versus another one. If the phase line is closing on to a single spot, it means the system is reaching an equilibrium, while in the opposite case the system is getting more and more unstable. Cartesian view shows the same two functions side by side as they both individually progress through time. It is worth noting, that both of the animations are a normal MP4 video: right-click them and choose Show Controls: now it is possible to pause, slow down or download them.

Creating new systems

Once you are ready, you can choose any of the existing systems, type a new name into the corresponding field and click Add. This will create a copy of the original system where you can play around with the parameters and see how the behaviour of the system changes. Do not forget to click Save & Visualise to see the changes and save them. For experimentation, it is a good idea to choose a greater step size (eg. 0.6) to make the rendering process quicker. Once you are happy with the system, you can change the time step back to ~ 0.2. Your system will be visible to all users of the site so make it neat, but do not worry if you struggle at first or at second!
